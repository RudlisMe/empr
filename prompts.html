Промт для ChatGPT
Ты — ChatGPT, помоги продолжить разработку проекта НКУ Automation. Важно: не проси заново проходить настройку Google Cloud, если я не говорю, что ключи/доступы потеряны. Уже сделано следующее:

1. Есть локальные Python-скрипты, которые можно запускать через консоль Windows/macOS:
   - fill_registry_nku_Onln_v9.py — заполнение онлайн-реестра НКУ в Google Sheets;
   - generate_passports_v7_Onln.py — генерация паспортов устройств по данным реестра;
   - passport_template_v2.docx — шаблон паспорта.

2. Дополнительно создан Telegram-бот для VPS. Он работает с той же Google Таблицей:
   - после /start просит ссылку на Google Таблицу;
   - проверяет доступ service account;
   - показывает меню: «Заполнять реестр» или «Сгенерировать паспорта»;
   - в режиме заполнения ведёт пользователя по шагам, близко к консольной логике;
   - в режиме генерации показывает последнюю строку данных, предлагает последние 5 строк или ввод начальной строки;
   - генерирует DOCX-паспорта и отправляет их в Telegram, при необходимости ZIP-архивом;
   - кнопка «❌ Отмена» должна корректно прерывать сценарий;
   - если после перезапуска старые кнопки Telegram не реагируют, рабочий сброс: /cancel и /start.

3. Рабочая папка на VPS: /opt/nku_bot. Бот запускается как systemd-служба nku-bot. Основные команды:
   - systemctl --no-pager status nku-bot
   - systemctl restart nku-bot
   - journalctl -u nku-bot -f
   - cd /opt/nku_bot && git status

4. Код сохранён в GitHub-репозитории:
   https://github.com/RudlisMe/nku-registry-bot
   В репозиторий должны попадать только код, шаблоны и инструкции. Нельзя добавлять:
   - .env
   - google_service_account.json
   - .venv/
   - __pycache__/
   - generated_passports/
   - архивы, резервные копии, токены и приватные ключи.

5. Для разворачивания на новом VPS подготовлен install_on_new_vps.sh. Команда установки из README:
   apt update && apt install -y curl git && curl -fsSL https://raw.githubusercontent.com/RudlisMe/nku-registry-bot/main/install_on_new_vps.sh -o /tmp/install_nku_bot.sh && REPO_URL=https://github.com/RudlisMe/nku-registry-bot.git bash /tmp/install_nku_bot.sh

6. При дальнейших правках соблюдай правила:
   - сохраняй совместимость локальных консольных скриптов Windows/macOS;
   - не меняй философию консольного ввода без отдельного согласования;
   - для новых версий файлов/скриптов добавляй новую версию в имени;
   - для обновлений, связанных с онлайн/Google Sheets, используй приписку Onln;
   - перед изменением логики сначала кратко опиши, какие файлы будут изменены;
   - после изменений дай точные команды для загрузки/обновления на VPS;
   - обязательно напомни проверить, что секреты не попали в git status.

Правила

Что учитывать при новых правках

Не ломать локальный режим

Локальные консольные скрипты для Windows/macOS остаются рабочим сценарием. Информацию о них в лендинге менять не нужно без отдельной задачи.

Не публиковать секреты

Файлы .env и google_service_account.json не должны попадать в GitHub, архивы для передачи или публичные инструкции.

Давать команды обновления

После каждой доработки нужны короткие команды: как загрузить файлы на VPS, перезапустить службу и проверить статус.
